# タイムシートウィンドウ方針 v1

## 目的

正式タイムシートUIは、作画中に常時参照する中核パネルである。
そのため、通常の右パネルやCellPanelへ押し込まず、独立した `TimesheetPanel` として扱う。

## 基本方針

- タイムシートUIは縦型にする。
- 縦軸はタイムラインT。
- 横軸はセル列。
- `CellPanel` には戻さない。
- 作画FとタイムラインTを同じ選択値で扱わない。

## 大ウィンドウ外へのドラッグ

ユーザー要望として、タイムシートウィンドウは可能ならソフト本体の大ウィンドウ外へドラッグできるようにする。

Dear ImGuiでは `ImGuiConfigFlags_ViewportsEnable` を有効にすると、バックエンド対応状況によってImGuiウィンドウをOSネイティブの別ウィンドウとして外へ出せる可能性がある。

ただし、このプロジェクトは現在 SDL3 + SDL_Renderer3 バックエンドを使用している。
外部ドラッグ対応は以下を確認してから行う。

1. `imgui_impl_sdl3` と `imgui_impl_sdlrenderer3` のmulti-viewports対応可否
2. SDL_Renderer3で複数OSウィンドウ描画が安定するか
3. WindowsのDPI、フォント、入力、IME、ドラッグ中の描画が壊れないか
4. メインウィンドウ終了時に外部タイムシートウィンドウが安全に閉じるか

## 実装優先度

Step 1ではまだUIを作らないため、外部ドラッグ機能は実装しない。

正式UI実装時の優先度は以下とする。

1. まずメインウィンドウ内の独立 `TimesheetPanel` として安定実装する。
2. 次に通常のImGuiフローティングウィンドウとしてドッキング・移動できるようにする。
3. 最後に、バックエンドが安定対応できる場合のみ、大ウィンドウ外へドラッグできるmulti-viewportsを有効化する。

## 判断

大ウィンドウ外へ出せる機能は便利だが、タイムシートのデータ意味、保存、表示解決、再生、出力より優先しない。

実装時に不安定なら、外部ドラッグはオプション設定にする。
