# Timesheet Rebuild Step 7.4.5: 中割ワークフローと背景参照設計

## 目的

Step 7.4 までで、タイムシートTを選んだときの表示解決結果を確認できるようにした。
次に出力系へ進む前に、中割と背景参照の設計を固定する。

理由は、アニメ制作では単に「中割」というラベルを付けるだけでは足りないからである。
自然な作業手順は次の通り。

1. 原画を先に作る。
2. 原画をタイムシート上のTへ置く。
3. 原画と原画の間に中割を作る。
4. 中割作画時は、前後の原画を見ながら描く。
5. 必要なら、背景・絵コンテ・レイアウトを下敷きとして置く。

## 用語整理

| 用語 | 意味 | UIでの使い方 |
| --- | --- | --- |
| T | タイムシート上の時間位置 | T1, T2, T3 |
| 作画F | 実際に描かれた絵の番号 | 作画F1, 作画F2 |
| 原画 | 作画Fの役割。動きの要点 | 役割: 原画 |
| 中割 | 前後原画の間を埋める作画F | 役割: 中割 / F1→F4の間 |
| コマ数 | その絵を何T分表示するか | 3コマ持ちなど |
| フレーム | 内部処理・レンダー・動画出力単位 | UI単独表記ではなるべく避ける |

## 中割の扱い

中割は自動生成画像ではない。
まずは「前後原画の間にある作画F」として扱う。

例:

```text
T1  キャラA 作画F1 役割: 原画
T4  キャラA 作画F3 役割: 中割  前原画F1 / 次原画F2 / 位置 1/2
T7  キャラA 作画F2 役割: 原画
```

表示・再生・出力では、原画も中割も同じく「指定された作画Fを表示する」。
違いは制作上の意味、ライトテーブル補助、確認UI、将来の中割支援に使う。

## 中割に必要なメタ情報

将来、作画Fまたはタイムシートentryに次のような情報を持たせる。

```text
role = Key / Inbetween / Drawing
previousKeyDrawingFrame = 作画F1
nextKeyDrawingFrame = 作画F2
inbetweenIndex = 1
inbetweenCount = 2
position = 1/3, 1/2 など
```

ただし、初期実装では保存形式を大きく変えず、まずは検出結果として扱う。

## 中割作成UIの方向

原画同士の間を選んだ状態で、次の操作を出す。

```text
この原画間に中割を1枚追加
この原画間に中割を2枚追加
この原画間に中割を3枚追加
```

例: T1とT7に原画があり、中割2枚を追加する場合:

```text
T1  原画 作画F1
T3  中割 作画F3
T5  中割 作画F4
T7  原画 作画F2
```

作画F番号は空き番号を自動採番する。
T位置は、原画間の空きTを均等に近い形で選ぶ。
既にentryがあるTは上書きせず、警告または別候補を出す。

## 中割作画時のライトテーブル

中割の編集対象を選んだ場合、自動で次を表示する。

```text
前原画: 薄く表示
次原画: 薄く表示
編集中の中割: 通常表示
```

ユーザーが手動でライトテーブルを毎回設定しなくてよい状態を目標にする。

## 背景・絵コンテ・レイアウト配置の考慮

今後、キャンバス背景に次のような素材を置けるようにする。

- 絵コンテ画像
- レイアウト
- 背景原図
- 仮背景
- 撮影参考
- キャラクター位置ガイド
- カメラフレーム/安全フレーム

これらはセル作画Fとは別の `参照/背景プレート` として扱う。
タイムシートのセル列に混ぜない。

## 参照/背景プレートの分類

| 種類 | 目的 | 出力 |
| --- | --- | --- |
| ReferenceOnly | 作画下敷き。絵コンテ、ラフ、資料 | 通常出力しない |
| PreviewOnly | プレビュー確認用。仮背景など | 設定により出力しない |
| RenderOutput | 完成背景や撮影素材 | 出力対象 |

## SceneResolverの将来形

タイムシート解決結果は、将来的に次の2系統を返す。

```text
1. animationCells
   タイムシートTで解決されたセル作画F

2. scenePlates
   背景、絵コンテ、レイアウト、カメラガイドなど
```

キャンバス表示では両方を重ねる。
PNG/MP4出力では `RenderOutput` のものだけ出力し、`ReferenceOnly` は出力しない。
